# UX Designer Skills for Claude Code

Expert UX design guidance powered by Laws of UX, Nielsen's Heuristics, WCAG, Figma best practices, and established design frameworks — without needing a UX team on call.

## Install

```
/plugin marketplace add https://github.com/trevorgrogers/ux-designer-skills
/plugin install ux-designer-skills
```

## What's Included

### 7 Specialist Agents

| Agent | What It Does |
|-------|-------------|
| **design-lead** | Main router — delegates to the right specialist |
| **design-critic** | Structured design critiques grounded in named principles |
| **interaction-designer** | Pattern selection, flows, states, micro-interactions |
| **accessibility-auditor** | WCAG 2.1/2.2 audits, ARIA patterns, inclusive design |
| **information-architect** | Navigation, taxonomy, content hierarchy, search |
| **design-systems-lead** | Tokens, components, documentation, governance |
| **usability-researcher** | Study design, test protocols, analysis, synthesis |

### 16 Skills (Knowledge Frameworks)

#### Foundations
| Skill | Frameworks |
|-------|-----------|
| **laws-of-ux** | All 30 Laws of UX — cognitive psychology applied to interface design |
| **design-heuristics** | Nielsen's 10 Heuristics + Shneiderman's 8 Golden Rules |
| **design-process** | IDEO Design Thinking, Double Diamond, Google Design Sprint, JTBD |
| **design-ethics** | Deceptive patterns, ethical design frameworks, GDPR/consent UX |

#### Craft
| Skill | Frameworks |
|-------|-----------|
| **design-critique** | Structured critique framework for giving and receiving feedback |
| **interaction-patterns** | Overlay, disclosure, input, navigation, and state patterns |
| **design-systems** | Token architecture, component specs, governance models |
| **motion-design** | Disney's 12 principles for UI, easing curves, duration guidelines |
| **mobile-design** | iOS HIG, Material Design 3, gestures, haptics, thumb zones |
| **modern-css** | Container queries, :has(), fluid typography, subgrid, view transitions |

#### Figma
| Skill | Frameworks |
|-------|-----------|
| **figma-best-practices** | File organization at scale, component architecture, variables/tokens, libraries, branching, performance, developer handoff |
| **figma-ai-coding** | Figma MCP for Claude Code/Cursor, Code Connect, vibe coding workflows, preparing files for AI, design-to-Tailwind pipelines |

#### Research & Measurement
| Skill | Frameworks |
|-------|-----------|
| **accessibility-wcag** | WCAG 2.1/2.2 quick reference, ARIA patterns, audit checklists |
| **information-architecture** | Navigation design, taxonomy, labeling, IA research methods |
| **usability-testing** | Study types, task writing, metrics, the 5-user rule |
| **ux-metrics** | Google HEART framework, SUS scoring, task success benchmarks |

## Usage

Just ask design questions naturally:

```
"Critique this checkout flow"
"Should I use a modal or a drawer for this?"
"Is this color contrast accessible?"
"How should I structure the navigation for this app?"
"Write a usability test plan for onboarding"
"How should I set up Figma for vibe coding with Claude Code?"
"What's the best way to organize Figma libraries at scale?"
"Prepare my Figma file for AI-powered implementation"
```
